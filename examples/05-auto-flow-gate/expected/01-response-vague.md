# 期望响应 · 输入 01(极度模糊)

> Agent 看到 inputs/01-vague.md 后**应该输出的东西**(参考样板)。

```
【需求 Gate · 完整性 ❌(0/11)】

我需要先把需求问清楚才能开工。请补充以下 11 项(每项一行):

1. 项目名 + 一句话价值主张
   → "项目叫什么?用一句话说清解决什么 / 给谁带来什么"
2. 用户人群
   → "谁会用?(独立开发者 / 团队 / 学生 / 老人 / 企业 / B 端)"
3. 核心场景(主流程 ≤ 3 步)
   → "用户从打开到完成主任务,3 步描述一下"
4. 平台/端
   → "Web / Mobile App / 小程序 / CLI / 桌面 / 后端 API,哪个?"
5. 技术栈偏好
   → "前端 React/Vue/原生?后端 Node/Python/Go?有硬约束吗?没偏好就明说"
6. 不做清单(防范围爬升)
   → "这一版明确不做的功能?(列 3-5 条)"
7. 验收标准(可验证)
   → "什么状态算'做完'?给出客观可测的标志,不是'好用'"
8. 部署目标
   → "部署到哪?(本地 / Vercel / CloudBase / 自有服务器)"
9. 时间预算
   → "希望几小时 / 几天内见到结果?'今晚'是多少小时?"
10. 外部依赖
    → "需要 API key / 设计稿 / 第三方账号 / 品牌资产 吗?谁提供"
11. 视觉/品牌方向(UI 项目)
    → "参考竞品 url + 期望氛围"

请一次性把 11 项都补齐 —— 中途插问会让需求碎片化。
```

## 关键点

- ❌ 不开工
- ❌ 不脑补
- ✅ 列出全部 11 项,每项配样板问句
- ✅ 明确说"一次补齐"避免来回拉扯
