# 访谈问题模板 + 共性识别方法

## 5 题深访模板(开放式)

```
1. 上次你遇到 <场景> 是什么时候?具体在做什么?
2. 当时怎么解决的?用了什么工具 / 找了谁 / 自己琢磨?
3. 整个过程里最让你不爽的是哪一步?为什么?
4. 你想象中"完美的解法"是什么样?(允许天马行空)
5. 如果有一个工具帮你做这件事,你愿意付费吗?多少?
```

第 5 题"愿付费"是**支付意愿测试**,但要轻看 —— 嘴上说付,真要付是另一回事。

## "为什么"问 3 次

```
访谈者:你为什么不喜欢现在的工具?
对方:太慢了
访谈者:慢在哪?(Why 1)
对方:每次要等 5 秒加载
访谈者:为什么 5 秒不能接受?(Why 2)
对方:我一天要操作 50 次,加起来 4 分钟没了
访谈者:这 4 分钟你拿来做什么?(Why 3)
对方:多回 2 个客户的消息 → 这才是真痛点
```

不问 3 次,只能拿到表面答案。

## 共性识别 3 步法

### Step 1 · 标签化
每个访谈记录,用 5-10 个标签标注:
```
访谈 #3: [频率高] [自己摸索] [想要快] [愿付费 50/月] [移动端为主]
```

### Step 2 · 标签频次表
```
| 标签 | 频次 |
|------|-----|
| 频率高 | 5/5 |
| 自己摸索 | 4/5 |
| 想要快 | 5/5 |
| 移动端为主 | 2/5 |
| 愿付费 | 3/5 |
```

### Step 3 · ≥3 人提到 = 共性
3+ = 共性洞察(写进报告)
1-2 = 异常洞察(单独标注,可能是早期信号)

## 不要问的问题(诱导 / 假设)

| 不要问 | 改成 |
|-------|------|
| "你觉得我们这个功能怎么样" | "上次你做 X 时是怎么处理的" |
| "你会不会用 Y 功能" | "上次遇到 Z 你用了什么工具" |
| "你需不需要 W" | "你最近一次做 W 是什么场景" |
| "AI 加进来好吗" | "你现在做这件事的瓶颈是什么" |

**意愿 ≠ 行为**,只问行为。
