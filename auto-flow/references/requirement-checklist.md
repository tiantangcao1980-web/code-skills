# 需求 Gate · 完整性 + 可行性详细清单

> Phase 0 的执行手册。**任一项缺失或冲突,绝不进 Phase 1**。

## 关卡 A · 完整性 11 项(必填)

### 1. 项目名 + 一句话价值主张
- **样板问句**:"项目叫什么?用一句话说清它解决什么 / 给谁带来什么"
- **不合格范例**:"做个工具帮我提效"(谁的什么效?提多少?)
- **合格范例**:"SnapMatte:给开发者拍截图自动加优雅留白与水印,5 秒搞定"

### 2. 用户人群
- **样板问句**:"谁会用?(独立开发者 / 团队 / 学生 / 老人 / 企业 / B 端)"
- **不合格**:"所有人"(等于没目标)
- **合格**:"独立开发者 + 5-20 人小团队"

### 3. 核心场景(主流程 ≤ 3 步)
- **样板问句**:"用户从打开到完成主任务,3 步描述一下"
- **合格**:"1. 拖图进窗口 → 2. 自动渲染留白 + 水印 → 3. 一键保存到剪贴板"

### 4. 平台 / 端
- **样板问句**:"哪个平台?(Web / Mobile App / 小程序 / CLI / 桌面 / 后端 API)"
- **合格**:"Web,响应式;暂不做移动 App"
- **特别注意**:小程序 ≠ Web,原生 App ≠ Hybrid,平台决定后续 90% 选型

### 5. 技术栈偏好
- **样板问句**:"前端 React/Vue/原生?后端 Node/Python/Go?有硬约束吗"
- **合格**:"React + TS + Tailwind;后端用 CloudBase 云函数;不接受 Vue"
- **没硬约束就明说**:"没偏好,选 AI 推荐"

### 6. 不做清单(范围爬升防火墙)
- **样板问句**:"**这一版明确不做的功能**?(列 3-5 条)"
- **不合格**:"做完之后再说"(没护栏)
- **合格**:"v0.1 不做:登录系统、付费、协作多人、批量处理"

### 7. 验收标准(可验证)
- **样板问句**:"什么状态算'做完'?给出**客观可测**的标志"
- **不合格**:"好用就行"
- **合格**:"打开 url 拖入 PNG,5 秒内生成结果图;移动端 Chrome / Safari 都能跑;首屏 <2s"

### 8. 部署目标
- **样板问句**:"部署到哪?"
- **选项**:本地跑 / Vercel / Netlify / CloudBase / GitHub Pages / 自有服务器 / Docker
- **影响**:静态托管 vs SSR vs Serverless 的取舍要在这一步定

### 9. 时间预算
- **样板问句**:"希望几小时 / 几天内见到结果?"
- **影响**:1 小时 → MVP 砍 80%;1 周 → 可以做完整 v0.1
- **不要含糊**:"越快越好"等于没说

### 10. 外部依赖
- **样板问句**:"需要 API key / 设计稿 / 第三方账号 / 品牌资产 吗?谁提供"
- **常见**:OpenAI key / 高德地图 / 微信开发者账号 / 域名 / 设计稿 Figma / Logo / 字体授权
- **不齐就停**:没设计稿别承诺 pixel-perfect

### 11. 视觉 / 品牌方向(UI 项目必填)
- **样板问句**:"参考竞品(贴 1-3 个 url)+ 期望氛围(瑞士主义 / 杂志风 / 赛博 / 极简 ...)"
- **不合格**:"看起来高级就行"
- **合格**:"参考 Vercel + Linear,深色 + 单色 + 一抹绿色,无炫光"

## 关卡 B · 可行性 4 项(一票否决)

### 1. 技术可行
- 用户期望的能力是否真存在?
- 例:"用纯前端做 PDF OCR" → 可行但慢,提示用 OCR 服务
- 例:"实时 4K 视频抠图,在浏览器跑" → 当前模型/带宽不行,改方案

### 2. 平台可行
- 平台限制是否冲突?
- 例:微信小程序 → 没有 `eval`、没有 Node、`<canvas>` API 受限
- 例:iOS App Store → 不能用 JIT、需要审核
- 例:CloudBase → 数据库类型决定 SDK 选择

### 3. 资源可行
- 外部依赖是否齐?
- 没设计稿 → pixel-perfect 不可能,只能给方向
- 没域名 → 不能配 HTTPS,只能给临时 URL
- 没账号 → 不能预付/部署

### 4. 时间可行
- 时间预算是否合理?
- "1 小时做完整电商" = NO;"1 小时做静态 landing" = YES
- 给出粗估:每幕大约花多久,总和 ≤ 用户预算

## Gate 输出格式(给用户看的需求快照)

```
【需求快照】

完整性 (11/11) ✅
- 项目: SnapMatte — 给开发者拍截图自动加留白与水印
- 用户: 独立开发者 + 小团队
- 主流程: 拖图 → 自动渲染 → 保存到剪贴板
- 平台: Web 响应式
- 技术: React + TS + Tailwind
- 不做: 登录 / 付费 / 协作 / 批量
- 验收: 5s 内出图,移动端可用,首屏 <2s
- 部署: Vercel
- 时间: 2 天
- 依赖: 无外部 API,Logo 我自己提供
- 视觉: 深色 + 单绿,Vercel/Linear 风格

可行性 (4/4) ✅
- 技术: ✅ Canvas API 足够
- 平台: ✅ Web 无限制
- 资源: ✅ Logo 用户提供
- 时间: ✅ 2 天充裕

---
请确认:回复"OK"开干,或指出要调整的项。
```

用户**明确说一句"OK"**(或"开始""开干""go"等同义),才进 Phase 1。

---

## 反模式

- ❌ "用户大概想要 X" 自己脑补,不问
- ❌ 11 项里只问到 7 项就开工
- ❌ 可行性看到红灯还硬上("应该能行吧")
- ❌ 输出快照后用户没明确确认就跑
- ❌ 中途又发现需求洞,再插进来一项 —— 应该一开始就问全
