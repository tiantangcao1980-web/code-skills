# Copywriting Design · 文案设计

> 灵感:[alchaincyf/huashu-design](https://github.com/alchaincyf/huashu-design) —— 文案不是翻译需求,是**人格体现**。
> 每个产品的文案要有**统一的"嘴型"**。

## 核心心法

**文案 = 一个虚构作者写给一类真实读者的话**。先确定作者人格,再写每句话。

---

## 语气四维度

每个维度选一个值,写下来,贴墙上。后续所有文案都要在这个坐标里:

| 维度 | 选项 |
|------|------|
| **正式度** | 学术 / 商务 / 朋友 / 街头 |
| **理性度** | 数据 / 逻辑 / 感性 / 诗意 |
| **温度** | 克制 / 中立 / 热情 / 煽情 |
| **幽默度** | 严肃 / 偶尔自嘲 / 频繁玩梗 / 玩世不恭 |

例:
- 开发者工具 → "朋友 + 逻辑 + 克制 + 偶尔自嘲"(Vercel、Linear)
- 母婴产品 → "朋友 + 感性 + 热情 + 严肃"
- 投资顾问 → "商务 + 数据 + 中立 + 严肃"

## 5 句关键文案规则

### 1. 首页主标题
- 一句话讲清"对谁有什么价值",**不是 slogan 比赛**
- ❌ "重新定义编程" ✅ "你不用记 git 命令了"

### 2. 主 CTA
- 动词开头 + 具体动作
- ❌ "立即体验" "了解更多" ✅ "免费导入第一个项目" "看 30 秒 demo"

### 3. 空态文案
- 不要 "暂无数据" "无内容"
- ✅ "还没人 X 过"、"创建第一个 X"、"导入示例数据看看"

### 4. 错态文案
- 说人话 + 告诉用户怎么办
- ❌ "操作失败" "Error 500" ✅ "保存没成功,网络好像断了。再试一次?"

### 5. 成功态文案
- 不要 "操作成功"
- ✅ 具体后果:"订单已发往杭州" "邮件已发给 12 位收件人"

## 工作流

### 触发场景 A · 产品起名 / slogan
1. 先定语气四维度
2. 列 10 个候选(不评判,先发散)
3. 用三个标准筛:
   - **5 秒能记住吗**?
   - **不被竞品已用**?
   - **能写到 logo 旁边不别扭**?
4. 留 3 个,问用户 / 做 A/B 测试

### 触发场景 B · 首页文案
1. 先定语气
2. 写主标题(讲清 who + what)
3. 写副标题(讲清 why now)
4. 写 CTA(主 + 次)
5. 写 social proof(数字 / 用户名 / logo)
6. 写常见疑问的预先回答(FAQ 头三个)

### 触发场景 C · 按钮 / 表单 / 错误文案逐条审查
按 5 句规则逐条 check,输出红黄绿报告。

## 文案禁用词清单

| 禁用 | 原因 |
|------|------|
| "极致" / "极速" / "超快" | 形容词堆砌,无信息 |
| "亲" / "哦" / "啦" | 装亲昵,显廉价(除非品牌就这定位) |
| "我们" / "本平台" | 公司视角,改成"你可以" |
| "立即体验" | 没有具体动作,等于没说 |
| "暂无数据" | 冷冰冰,改成有人格的空态 |
| "操作成功 / 失败" | 不告诉用户具体是什么 |
| 行业黑话(BD/PV/UV) | 除非用户就是行业人 |

---

## 反模式

- ❌ **形容词堆砌**:"极致流畅、超快速度、强力性能"
- ❌ **公司视角**:"我们提供" → "你可以"
- ❌ **CTA 模糊**:"立即体验" → "免费导入第一个项目"
- ❌ **多个语气混用**:首页严肃,按钮玩梗,错态又装亲 → 精分
- ❌ **没人格**:看上去和竞品 99% 像
- ❌ **抄竞品**:抄过来不一定适合你的语气坐标
