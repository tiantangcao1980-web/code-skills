# 设计品味整合

> 融合三个项目的精华:
> - [Leonxlnx/taste-skill](https://github.com/Leonxlnx/taste-skill) — 审美维度的判定
> - [alchaincyf/huashu-design](https://github.com/alchaincyf/huashu-design) — 文案 / 话术 / 文字感
> - [tiantangcao1980-web/awesome-design-md](https://github.com/tiantangcao1980-web/awesome-design-md) — 设计规范模板

本文件不是设计教程(那是 `ui-design` skill 的事)。
本文件是**判定标准**:怎么判断一份设计规范是否过关。

## 一、Aesthetic Direction(美学方向)

### 不接受的方向词

| 含糊词 | 为什么不行 |
|--------|-----------|
| "现代简约" | 99% 的项目都自称这个 |
| "干净优雅" | 描述不出任何具体特征 |
| "高级感" | 主观且无法验证 |
| "符合 Z 世代审美" | Z 世代审美差异极大,等于没说 |
| "扁平化" | 2014 年的词,且不指导具体决策 |

### 接受的方向词(参考)

| 方向 | 关键特征 | 适合场景 |
|------|---------|---------|
| 瑞士主义 / 国际主义 | 网格、Helvetica/Inter、强对比、留白多 | 工具产品、文档站 |
| 杂志风 / Editorial | 衬线大字、不对称、多层信息 | 内容平台、长文 |
| 极客终端风 / Brutalist | 等宽字、ASCII 边框、单色 | 开发者工具、CLI |
| 苹果风 / Glass UI | SF Pro、毛玻璃、圆角、渐变 | 消费应用 |
| 站酷新国潮 / 中式 | 宋黑搭配、印章、留白、素雅 | 国风内容 |
| 杜尚式反美学 | 故意丑、错位、噪点 | 艺术项目、抢眼营销 |
| 工业终端风 / Aviation | 仪表盘、数字字体、深色背景 | 监控、Dashboard |

**进度门**:能用上面任一方向 + 一个具体形容词描述自己的项目。
例:"瑞士主义 + 但字号比典型瑞士派大一档,因为是给 50 岁用户看的"。

## 二、配色判定

### 必填字段
- **主色**:大色块、按钮、强调
- **辅色 1-2**:点缀、状态标识
- **背景色**:页面底色(可能多层)
- **文本色**:正文、次要、禁用三档
- **状态色**:success / warning / error / info(各一个)

### 红线
- ❌ 不要 5 个以上主色调
- ❌ 不要 70% 灰色 + 30% 蓝色这种"安全配色"(没记忆点)
- ❌ 不要在按钮上用纯黑(`#000`)或纯白(`#fff`),用 `#0a0a0a` / `#fafafa` 这种偏移色

### 验证手段
- 跑 contrast checker(文本/背景对比度 WCAG AA 起步)
- 色盲模拟器(deuteranopia / protanopia)看是否仍可用
- 灰度模式看(信息层级是否仍清晰)

## 三、字体判定

### 必填
- 标题字体(具体名字,不是 "sans-serif")
- 正文字体(具体名字)
- 等宽字体(若有代码 / 数字展示)

### 推荐组合参考
| 标题 | 正文 | 风格 |
|------|------|------|
| Inter | Inter | 通用现代 |
| Geist | Geist Mono | 极客 / 开发者 |
| Playfair Display | Source Serif | 杂志 / 内容 |
| 思源黑体 / Noto Sans SC | 思源宋体 | 中文项目 |
| SF Pro Display | SF Pro Text | iOS / Apple 风 |
| IBM Plex Sans | IBM Plex Mono | 工程感 |

### 红线
- ❌ 中文项目用 Helvetica(中文 fallback 通常很丑)
- ❌ 同时用 3+ 字体家族(臃肿)
- ❌ 标题正文都用同一字号(没层级)

## 四、布局策略

### 拒绝模板
- ❌ 顶部 logo + 主菜单 + 居中 hero + 三列 features 卡片(默认 AI 生成的样子)
- ❌ 全屏背景图 + 正中大字 + 一个按钮(没信息含量)

### 鼓励的策略
- 不对称网格(2/3 + 1/3)
- 大尺度标题(超过 80px)
- 内容溢出可视区域(暗示"还有更多")
- 文字与图片错位(打破对齐感)
- 强留白(关键信息周围 80px+ 空)

**进度门**:布局策略要能用一句话说清,例:"主页面采用 2/3 + 1/3 网格,左侧是大标题,右侧是关键产品卖点轮播,留白集中在右下"。

## 五、文案语气(huashu-design 借鉴)

文案不是"翻译需求",文案是**人格体现**。每个项目的文案要有**统一的"嘴型"**。

### 语气维度
- **正式度**:学术 / 商务 / 朋友 / 街头
- **理性度**:数据 / 逻辑 / 感性 / 诗意
- **温度**:克制 / 中立 / 热情 / 煽情
- **幽默度**:严肃 / 偶尔自嘲 / 频繁玩梗 / 玩世不恭

### 关键文案 5 句必填
| 场景 | 注意事项 |
|------|---------|
| **首页主标题** | 一句话讲清"对谁有什么价值",不是 slogan 比赛 |
| **主 CTA** | 动词开头,具体动作,不要"立即体验" |
| **空态** | 不要"暂无数据",改成"还没人 X 过" |
| **错态** | 说人话,告诉用户怎么办 |
| **成功态** | 不要"操作成功",改成具体后果("订单已发往 杭州") |

### 文案反模式
- ❌ 形容词堆砌:"极致流畅、超快速度、强力性能"
- ❌ 公司视角:"我们提供"、"我们的产品",改成"你可以"
- ❌ 行业黑话堆砌(除非用户就是行业人)

## 六、设计规范文档模板(awesome-design-md 借鉴)

最终输出一份 `docs/DESIGN.md`,模板:

```md
# <项目名> Design Specification

## Aesthetic Direction
- 主方向: <例如 "瑞士主义">
- 局部偏移: <例如 "字号偏大,因受众老龄">
- 参考:[图 1](url) / [产品 X](url)

## Color
| Token | Value | Usage |
|-------|-------|-------|
| color/primary | #1e293b | 主按钮、Logo |
| ... | ... | ... |

## Typography
| Token | Family | Weight | Size |
|-------|--------|--------|------|
| heading/h1 | Geist | 700 | 64/72 |
| body/default | Geist | 400 | 16/24 |
| ... | ... | ... | ... |

## Spacing & Radius
- 间距步长: 4px
- 圆角: 0 / 4 / 8 / 16 (业务里只用这 4 个)

## Layout
- 主网格: 12 列,gutter 24px
- 主页面: 2/3 + 1/3 不对称
- 移动端: 单列堆叠,padding 16px

## Motion
- 缓动: `cubic-bezier(0.4, 0, 0.2, 1)`
- 时长: 150ms (微交互) / 300ms (页面级) / 不超过 500ms

## Voice & Tone
- 语气: <一句话>
- 关键文案 5 句: <填入>
- 禁用词: <列表,例如 "亲""哦"等>

## Accessibility
- 对比度: WCAG AA(正文 4.5:1,大字 3:1)
- 焦点态: 必须可见
- 键盘可达: 所有交互
```

## 七、taste-skill 借鉴:审美自检清单

> 每条只问 yes / no,no 多于 3 条 = 设计规范不过关。

- [ ] 看 5 秒能记住一个视觉特征?
- [ ] 把 logo 遮住,能从字体/配色/布局认出是同一品牌系列?
- [ ] 关键文案没有形容词堆砌?
- [ ] 没有"我们""极致""一站式"这种词?
- [ ] 主页面**不是**居中三列卡片?
- [ ] 主色不是蓝色 + 灰色?(可以是,但要有理由)
- [ ] 字体不是 Helvetica + Arial 的默认?
- [ ] 有至少一处"不那么常见"的细节(独特图标 / 微动效 / 文字游戏)?
- [ ] 在黑白打印下信息层级仍然清晰?
- [ ] 移动端不是简单缩小,而是有移动专属布局?
